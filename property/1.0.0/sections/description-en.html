Hydrogen Valley Ontology (H2V) - Property module: Description back to ToC

The Property module of the Hydrogen Valley Ontology defines all the properties, both dynamic and static, that can be associated with the various HV systems, as gathered from the collected requirements. For each property, a specific subclass of StaticProperty or DynamicProperty is defined, along with a specific subproperty of hasProperty. For instance, for the static parameter indicating the electric power range of an electrolyzer, there is the class ElectricPowerRange, defined as a subclass of StaticProperty, and a property hasElectricPowerRange to connect an Electrolyzer to its ElectricPowerRange. Furthermore, for each static and dynamic property, OWL property restrictions are used to specify the applicable measurement unit (applicableUnit) based on our requirements, such as KiloW, i.e., Kilowatt, in the case of ElectricPowerRange. The measurement units are reused from the ["Unit" vocabulary associated with QUDT](https://www.qudt.org/doc/DOC_VOCAB-UNITS.html). Lastly, the property is also linked, via OWL property restrictions, to the more general reference QuantityKind (from the ["Quantity Kinds" vocabulary associated with QUDT](https://www.qudt.org/doc/DOC_VOCAB-QUANTITY-KINDS.html)), which represents an abstract type of physical quantity, not specific to a particular system, for example, ElectricPower in the case of ElectricPowerRange. Multiple properties can be associated with the same QuantityKind. The module includes 46 static properties and 49 dynamic properties.
