Hydrogen Valley Ontology (H2V) - Property module: Description back to ToC

The Property module of the Hydrogen Valley Ontology defines all the properties, both dynamic and static, that can be associated with the various HV systems, as gathered from the collected requirements. For each property, a specific named individual of StaticProperty or DynamicProperty is defined. For instance, for the static parameter indicating the maximum electric power of an electrolyzer, there is the named individual [MaximumElectricPower](https://w3id.org/h2v/property#MaximumElectricPower) of the class StaticProperty. Such individual is linked to the class representing the system it applies to (via the property appliesToSystem), in this case, the Electrolyzer class. Furthermore, for each static and dynamic property, we specify the applicable measurement unit (applicableUnit) based on our requirements, such as KiloW, i.e., Kilowatt, in the case of MaximumElectricPower. The measurement units are reused from the ["Unit" vocabulary associated with QUDT](https://www.qudt.org/doc/DOC_VOCAB-UNITS.html). The property is also linked, via skos:broader, to the more general reference QuantityKind (from the ["Quantity Kinds" vocabulary associated with QUDT](https://www.qudt.org/doc/DOC_VOCAB-QUANTITY-KINDS.html)), which represents an abstract type of physical quantity, not specific to a particular system, for example, ElectricPower in the case of the MaximumElectricPower of an electrolyzer. Multiple properties can be associated with the same QuantityKind. Finally, when it comes to properties that can be grouped based on some purpose, as in the case of a range of acceptable values, we use the class PropertySet, which is linked to its member properties with the hasPropertyMember property, as its subproperties (e.g., hasOptimalPropertyMember). The module includes 125 static properties and 75 dynamic properties (3 of which are weather properties).
