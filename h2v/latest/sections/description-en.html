Hydrogen Valley Ontology (H2V): Description back to ToC

The main class of the ontology is [HydrogenValley](https://w3id.org/h2v#HydrogenValley), defined as a geographical area where various hydrogen applications are combined into an integrated ecosystem, and which can cover the entire hydrogen value chain. A HydrogenValley is both a system (declared as a subclass of [System](http://www.w3.org/ns/ssn/System)) and a (Geographical)Feature, being connected to a geometry (Geometry) through the property hasGeometry. The different areas constituting the hydrogen value chain are represented as subsystems (hasSubSystem) of the HydrogenValley via the class HydrogenValleyArea, and its subclasses [HydrogenProductionArea](https://w3id.org/h2v#HydrogenProductionArea), [HydrogenConsumptionArea](https://w3id.org/h2v#HydrogenConsumptionArea), [HydrogenStorageArea](https://w3id.org/h2v#HydrogenStorageArea), and [ElectricityProductionArea](https://w3id.org/h2v#ElectricityProductionArea). In turn, a HydrogenValleyArea can have subsystems, namely [HydrogenValleyComponents](https://w3id.org/h2v#HydrogenValleyComponent). However, not all HydrogenValleyComponents can be traced back to a HydrogenValleyArea: in this case, the HydrogenValley will be directly connected to the HydrogenValleyComponent with the property hasSubSystem. All systems constituting a HydrogenValley, including the HydrogenValley itself, can be associated with a Geometry, which allows defining the spatial characteristics of the system based on the type of geometry (polygon, point, etc.), with an identifier (property identifier), a name (property title), and a description (property description). The ontology also defines all components that can belong to a HydrogenValley, as subclasses of System: - [Electrolyzer](https://w3id.org/h2v#Electrolyzer) to represent the electrolyzer, that is a system that uses electrical energy to split water into hydrogen and oxygen; - [RESPlant](https://w3id.org/h2v#RESPlant) to represent a renewable energy plant that produces energy for the electrolyzer, and [PhotovoltaicSystem](https://w3id.org/h2v#PhotovoltaicSystem), i.e., a photovoltaic plant, as a specific type of RESPlant; - [HydrogenStorage](https://w3id.org/h2v#HydrogenStorage) to represent the hydrogen storage system; - [Pipe](https://w3id.org/h2v#Pipe) to represent the pipes for transporting hydrogen within the HV; - [PointDevice](https://w3id.org/h2v#PointDevice) to represent the device for the exchange of energy vectors in general, and [FuelCell](https://w3id.org/h2v#FuelCell), that converts the chemical energy of a fuel (e.g., hydrogen) into electricity, and [Blender](https://w3id.org/h2v#Blender), that is the transfer point with the gas network, as specific types of PointDevices; - [HydrogenConsumer](https://w3id.org/h2v#HydrogenConsumer) to represent the system that utilizes the produced hydrogen for various purposes; - [Valve](https://w3id.org/h2v#Valve) to represent a device for controlling the passage of fluid or air through a pipe, and specific types of valves, that is: ActuatedOn-OffValve, ManualValve, PressureControlValve, and GlobeValve; - [WaterTank](https://w3id.org/h2v#WaterTank) to represent a storage system for distilled water used as water source for the electrolyzer; - [WaterPump](https://w3id.org/h2v#WaterPump) to represent a system that allows water flow from water the tank to the electrolyzer. Each system present in the HydrogenValley can be associated with one or more properties. Within the ontology, two types of properties are distinguished: static and dynamic. A StaticProperty is a nominal property of a system, typically defined by design or in the manufacturer's specifications, and is not intended to be dynamically observed or measured (e.g., the range of minimum and maximum electrical power allowed by a certain electrolyzer). A DynamicProperty (defined in the ontology as a subclass of ObservableProperty) is instead a property of a system that can vary over time and is dynamically observed or measured (e.g., the actual electrical power of an electrolyzer as measured at a certain time). A System is linked to a PropertyOfInterest, that is a property tied to a specific real-world System, which is in turn linked to its reference kind, that is a [StaticProperty](https://w3id.org/h2v#StaticProperty), which is instead independent from real-world systems. The PropertyOfInterest is directly associated with a QuantityValue through the property hasPropertyValue. A QuantityValue is a numerical value associated with a physical quantity, expressed in a specific unit of measure (property hasUnit, with codomain Unit). In the case of StaticProperty, a single value (numericValue) is defined. The [PropertySet](https://w3id.org/h2v#PropertySet) allows us to group properties for a specific purpose. These properties are usually members of a range (maximum/minimum), with a possible optimal value within the range. In the case of a [DynamicProperty](https://w3id.org/h2v#DynamicProperty), the numerical value is not predefined but is measured and thus inserted in the context of a specific Observation. The Observation is connected to the DynamicProperty being observed (observedProperty), the specific system to which the observed property is associated (hasFeatureOfInterest), and the QuantityValue resulting from the observation (hasResult). The QuantityValue will therefore include both the measured numerical value (numericValue) and the unit of measure used (hasUnit). The ontology models also 3 types of activities that can be relevant for an HydrogenValley: - [Forecast](https://w3id.org/h2v#Forecast), to represent an activity that produces a prediction about the future value of some Property (in this case, the DynamicProperty is inserted in the context of a specific Forecast), and, more specifically, [WeatherForecast](https://w3id.org/h2v#WeatherForecast), to represent an activity that produces a prediction about the future value of some WeatherProperty, that is a subclass of DynamicProperty; - [HydrogenAllocation](https://w3id.org/h2v#HydrogenAllocation) to represent an activity in which an amount of hydrogen is allocated by a System, usually an Hydrogen Valley, to another System; - [HydrogenImportExport](https://w3id.org/h2v#HydrogenImportExport) to represent an activity in which an amount of hydrogen is exchanged between two systems, that is, an hydrogen importer and an hydrogen exporter. It is also possible to model the [TechnicalDrawing](https://w3id.org/h2v#TechnicalDrawing) graphically representing an HydrogenValley, and to link the Geometry of all its components within the drawing with the property inTechnicalDrawing. Finally, in order to represent the topology of the HydrogenValley, the ontology introduces the class [Terminal](https://w3id.org/h2v#Terminal), which is linked to an HV Component and represents a connection point of that system to a piece of pipe. A terminal, or multiple terminals, of an HV Component can be included in a [ConnectivityNode](https://w3id.org/h2v#ConnectivityNode), meaning that all terminals in a certain connectivity node are physically connected together.

Usage examples

The following example shows how an hydrogen valley can be linked to its hydrogen valley areas and hydrogen valley components, using the relation hasSubSystem. ``` ex:HV01 rdf:type h2v:HydrogenValley ; ssn:hasSubSystem ex:HV01-production . ex:HV01-production rdf:type h2v:HydrogenProductionArea ; ssn:hasSubSystem ex:E01 . ex:E01 rdf:type h2v:Electrolyzer , h2v:HydrogenValleyComponent . ``` The following example shows the usage of PropertyOfInterest for specifying the value of a certain property (e.g., the maximum electric power that can be absorbed by the electrolyzer without compromising its integrity) for a specific real-world system, e.g., an electrolyzer. ``` ex:E01 rdf:type h2v:Electrolyzer , h2v:HydrogenValleyComponent . ex:E01 saref:hasPropertyOfInterest ex:E01-maximum-electric-power . ex:E01-maximum-electric-power saref:hasPropertyKind h2v-prop:MaximumElectricPower ; h2v:hasPropertyValue ex:400-kw . ex:400-kw rdf:type qudt:QuantityValue ; qudt:numericValue "400"^^xsd:double ; qudt:hasUnit unit:KiloW . h2v-prop:MaximumElectricPower rdf:type h2v:StaticProperty . ``` The following example shows the usage of Observation to represent a measurement of a certain dynamic property (e.g., the electric power absorbed by the electrolyzer at a certain time instant). This value can be compared with the static property maximum electric power to check if the electrolyzer is working within its limits. ``` ex:E01 rdf:type h2v:Electrolyzer , h2v:HydrogenValleyComponent . ex:E01-measurement-2025-10-01 rdf:type sosa:Observation ; qudt:numericValue "450"^^xsd:double ; qudt:hasUnit unit:KiloW ] . h2v-prop:ElectricPower rdf:type h2v:DynamicProperty . ``` The following example shows the Forecast of a certain dynamic property (e.g., the module temperature of a photovoltaic system at a certain future time instant, equal to 30 degrees Celsius). ``` ex:PV01 rdf:type h2v:PhotovoltaicSystem , h2v:HydrogenValleyComponent . ex:PV01-forecast-2025-10-01 rdf:type h2v:Forecast ;
